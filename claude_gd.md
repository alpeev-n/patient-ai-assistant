# Claude Code — практическое руководство

## 1. Перед запуском

Убедитесь что вы в нужной директории проекта и в git-репозитории:

```bash
cd my-project
git init
git status
```

Настройте ограничения для проекта (опционально, но рекомендуется).
Создайте файл `.claude/settings.json`:

```json
{
  "permissions": {
    "allow": ["Read", "Grep", "Write"],
    "deny": ["Bash"]
  }
}
```

Или запустите с флагами ограничений напрямую:

```bash
claude --allowedTools "Read,Grep,LS"       # только чтение
claude --disallowedTools "Bash"            # запрет выполнения команд
```

---

## 2. Запуск сессии

```bash
claude                                     # интерактивная сессия
claude --model claude-sonnet-4-6           # выбрать модель явно
claude --system "Ты senior Python разработчик, пиши лаконичный код"
```

При старте Claude видит структуру вашего проекта автоматически.

---

## 3. В начале сессии

Сориентируйте Claude в контексте задачи:

```
> Мы работаем над FastAPI-проектом. Сегодня нужно добавить авторизацию через JWT.
  Вот структура проекта: [опишите или попросите Claude прочитать нужные файлы]
```

Проверьте модель и настройки:

```
/model          — текущая модель, можно сменить прямо здесь
/config         — все настройки сессии
```

---

## 4. Во время работы

### Контроль над действиями Claude

Когда появляется запрос `This command requires approval`:

- **`1. Yes`** — разрешить один раз (рекомендуется по умолчанию)
- **`2. Yes, and don't ask again`** — только если полностью доверяете действию
- **`3. No`** — отказать, если что-то смущает

Всегда читайте что именно Claude собирается выполнить перед подтверждением.

### Мониторинг токенов и контекста

```
/context        — сколько токенов использовано и сколько осталось
/cost           — стоимость текущей сессии (для API-пользователей)
/stats          — статистика (для Pro/Max подписчиков)
```

### Смена модели на лету

```
/model claude-opus-4-6      — для сложных архитектурных задач
/model claude-sonnet-4-6    — для обычной работы (по умолчанию)
/model claude-haiku-4-5     — для простых и быстрых задач
```

### Если контекст переполняется

```
/compact                                        — сжать историю автоматически
/compact Focus on code changes and test output  — сжать с приоритетом
```

После `/compact` часть деталей теряется — используйте когда завершили крупный блок работы.

## 5. Безопасность — страховочная сетка

Перед любыми крупными изменениями делайте коммит:

```bash
git add .
git commit -m "before claude session: add auth feature"
```

После работы проверяйте что изменилось:

```bash
git diff                  # все изменения
git diff src/auth.py      # конкретный файл
```

Откат если что-то пошло не так:

```bash
git checkout .            # откатить все несохранённые изменения
git revert HEAD           # отменить последний коммит
```

## 6. Работа с несколькими задачами

Для каждой отдельной задачи — новая сессия. Это экономит токены и сохраняет контекст чистым.

Перед завершением сессии переименуйте её:

```
/rename auth-jwt-implementation
```

Чтобы вернуться к ней позже:

```bash
claude --continue         # продолжить последнюю сессию
```

```
/resume                   — выбрать сессию из списка
```

## 7. Полезные сценарии в одну строку (без интерактивной сессии)

```bash
# Code review перед коммитом
git diff | claude -p "Сделай code review, найди баги и проблемы"

# Автогенерация commit message
git diff --staged | claude -p "Напиши git commit message"

# Анализ ошибки сборки
npm run build 2>&1 | claude -p "Объясни ошибку и как исправить"

# Генерация тестов
claude -p "Напиши unit-тесты для всех функций" < utils.py > test_utils.py

# Генерация документации
claude -p "Напиши JSDoc для всех функций" < api.js
```

## 10. Быстрая шпаргалка

| Команда | Когда использовать |
|---|---|
| `claude` | Начать новую сессию |
| `claude -p "Объясни этот код"` |
| `claude -p "Что делает этот файл?" < main.py` |
| `cat app.js claude -p` "Найди баги в этом коде" |
| `/model` | Сменить модель под задачу |
| `/context` | Проверить расход токенов |
| `/cost` | Проверить стоимость сессии |
| `/compact` | Контекст переполняется |
| `/rename` | Перед завершением сессии |
| `/resume` | Вернуться к старой сессии |
| `claude --continue` | Продолжить последнюю сессию |
| `git diff` | Проверить что изменил Claude |
| `git checkout .` | Откатить нежелательные изменения |
